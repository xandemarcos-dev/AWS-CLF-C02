# Amazon S3 (Simple Storage Service)

## Em uma frase
> Armazenamento de **objetos** infinitamente escalável e altamente durável.

## O que é / pra que serve
Guarda arquivos (objetos) em **buckets**. Backups, sites estáticos, data lakes,
armazenamento de mídia. 99,999999999% (11 noves) de durabilidade.

## Quando usar (e quando NÃO usar)
- ✅ Arquivos, backups, sites estáticos, dados não estruturados, qualquer volume.
- ❌ Disco de uma instância EC2 → [[EBS]]. Sistema de arquivos compartilhado → [[EFS]].

## Conceitos-chave
- **Objeto** (arquivo + metadados) dentro de **bucket** (nome global único).
- **Classes de armazenamento** (ver preço).
- **Versionamento**, **criptografia** ([[KMS]]) e políticas de acesso ([[IAM]]/bucket policy).
- **Lifecycle rules:** mover objetos automaticamente para classes mais baratas.

## 💰 Modelo de preço
- Paga por: **armazenamento**, **requisições** e **transferência de saída**.
- Classes: **Standard** (acesso frequente) · **Standard-IA / One Zone-IA** (acesso raro) ·
  **Glacier / Glacier Deep Archive** (arquivamento, recuperação mais lenta e barata).
- **Intelligent-Tiering:** move automaticamente conforme o padrão de acesso.

## ⚠️ Pegadinhas de prova
- S3 = armazenamento de **objetos** (não é sistema de arquivos nem disco de bloco).
- **Glacier** = arquivamento barato e recuperação demorada.
- Nome do bucket é **globalmente único**.
- Durabilidade (11 noves) ≠ disponibilidade.
