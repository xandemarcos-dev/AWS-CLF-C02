Erros mais comuns na prova

Pegadinhas clássicas que derrubam quem estudou mas não prestou atenção nos detalhes.

⚠️
"Responsabilidade compartilhada" tem pegadinhas
AWS gerencia hardware, hypervisor e infraestrutura física. VOCÊ gerencia SO, patches, dados, IAM e rede dentro da VPC. Borda cinza: RDS — AWS gerencia o SO do banco; você gerencia dados e acesso.
⚠️
IAM Roles ≠ IAM Users
User tem credenciais permanentes. Role é assumida temporariamente (EC2 acessa S3 via role, não via usuário). Sempre prefira roles para serviços AWS.
⚠️
S3 não é sistema de arquivos
S3 = object storage. EBS = block storage (disco de VM). EFS = file system compartilhado (NFS). Confundir esses três é erro clássico.
⚠️
High Availability ≠ Fault Tolerance
HA = minimiza downtime (pode ter breve interrupção). FT = zero downtime. Multi-AZ = HA. Active-Active multi-região = FT.
⚠️
CloudWatch vs CloudTrail
CloudWatch = métricas e logs (CPU, alarmes). CloudTrail = auditoria de API (quem fez o quê). "Quem deletou o bucket?" = CloudTrail.
⚠️
Reserved Instances vs Savings Plans
Reserved = compromisso com instância específica. Savings Plans = compromisso com gasto em $/hora (mais flexível). Savings Plans é mais moderno.
