Erros mais comuns na prova

Pegadinhas clássicas que derrubam quem estudou mas não prestou atenção nos detalhes.

⚠️
"Responsabilidade compartilhada" tem pegadinhas
A AWS gerencia o hardware físico, hypervisor e infraestrutura. VOCÊ gerencia SO, patches, dados, IAM, configuração de rede dentro da VPC. Borda cinza: RDS — AWS gerencia o SO e patches do banco; você gerencia dados e acesso.
⚠️
IAM Roles ≠ IAM Users
User tem credenciais permanentes. Role é assumida temporariamente (ex: EC2 acessa S3 via role, não via usuário). Na prova aparecem cenários de "qual é a forma correta" — sempre prefira roles para serviços.
⚠️
S3 não é sistema de arquivos
S3 é object storage. EBS é block storage (disco de VM). EFS é file system compartilhado (NFS). Confundir esses três é erro clássico.
⚠️
High Availability ≠ Fault Tolerance
HA = minimize downtime (pode ter breve interrupção). FT = zero downtime (continua operando mesmo com falha). Multi-AZ = HA. Active-Active multi-região = FT.
⚠️
CloudWatch vs CloudTrail
CloudWatch = monitoramento de MÉTRICAS e LOGS (CPU, memória, alarmes). CloudTrail = AUDITORIA de chamadas de API (quem fez o quê, quando). Pergunta "quem deletou o bucket?" = CloudTrail.
⚠️
Reserved Instances vs Savings Plans
Reserved = compromisso com instância específica. Savings Plans = compromisso com gasto em dólar/hora (mais flexível). Ambos dão desconto, mas Savings Plans é mais moderno.
